Guia de Configuração

Siga estes passos para colocar o Portal Pesquisadoras ES no ar. Tempo estimado: 30 minutos.

1
Criar conta gratuita no Supabase

O Supabase é o serviço que vai guardar todos os dados do portal (pesquisadoras, publicações, blog). É gratuito para projetos pequenos.

Acessar supabase.com →
Clique em Start your project e crie uma conta com seu e-mail ou com o Google. Depois clique em New project.
  • ✓
    Dê o nome: pesquisadoras-es
  • ✓
    Anote a senha do banco de dados que você criar (guarde em local seguro)
  • ✓
    Escolha a região: South America (São Paulo)
  • ✓
    Clique em Create new project e aguarde ~2 minutos
2
Copiar as chaves de acesso

Dentro do painel do Supabase, vá em Settings → API. Você vai ver duas informações importantes:

ItemOnde encontrarO que você vai copiar
Project URLSettings → API → Project URLComeça com https:// e termina com .supabase.co
Anon KeySettings → API → Project API Keys → anon publicString longa de letras e números
⚠ Nunca compartilhe a chave service_role. Use apenas a anon/public.
3
Configurar o arquivo config.js

Abra o arquivo config.js (que está na pasta do portal) e substitua os valores:

// Substitua os valores abaixo pelos seus: const SUPABASE_URL = 'https://SEU-PROJETO.supabase.co'; const SUPABASE_ANON_KEY = 'sua-anon-key-aqui'; // Chave da Anthropic para o cadastro com IA (opcional): const ANTHROPIC_API_KEY = 'sk-ant-...';
💡 A chave da Anthropic é usada na funcionalidade de Cadastro com IA. Se não quiser usar essa função, pode deixar em branco.
4
Criar as tabelas no banco de dados

No painel do Supabase, vá em SQL Editor e cole o código abaixo. Depois clique em Run.

-- TABELA: Pesquisadoras CREATE TABLE researchers ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, nome TEXT NOT NULL, mini_bio TEXT, area_principal TEXT, instituicao TEXT, lattes TEXT, orcid TEXT, linkedin TEXT, google_scholar TEXT, research_gate TEXT, email TEXT, site TEXT, status TEXT DEFAULT 'pendente', -- pendente | aprovada | reprovada destaque BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT NOW() ); -- TABELA: Publicações CREATE TABLE works ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, researcher_id UUID REFERENCES researchers(id), titulo TEXT NOT NULL, tipo TEXT, ano INTEGER, resumo TEXT, area_conhecimento TEXT, instituicao TEXT, tags TEXT[], link_acesso TEXT, link_compra TEXT, isbn TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); -- TABELA: Blog CREATE TABLE blog_posts ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, titulo TEXT NOT NULL, resumo TEXT, conteudo TEXT, categoria TEXT DEFAULT 'Artigo', emoji TEXT DEFAULT '✍️', autor TEXT, publicado BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT NOW() ); -- PERMISSÕES: Leitura pública (necessário) ALTER TABLE researchers ENABLE ROW LEVEL SECURITY; ALTER TABLE works ENABLE ROW LEVEL SECURITY; ALTER TABLE blog_posts ENABLE ROW LEVEL SECURITY; CREATE POLICY "Leitura pública researchers" ON researchers FOR SELECT USING (true); CREATE POLICY "Inserção pública researchers" ON researchers FOR INSERT WITH CHECK (true); CREATE POLICY "Leitura pública works" ON works FOR SELECT USING (true); CREATE POLICY "Inserção pública works" ON works FOR INSERT WITH CHECK (true); CREATE POLICY "Leitura pública blog" ON blog_posts FOR SELECT USING (true); -- ADMIN: Permissões totais para usuários autenticados CREATE POLICY "Admin researchers" ON researchers FOR ALL USING (auth.role() = 'authenticated'); CREATE POLICY "Admin works" ON works FOR ALL USING (auth.role() = 'authenticated'); CREATE POLICY "Admin blog" ON blog_posts FOR ALL USING (auth.role() = 'authenticated');
5
Criar o usuário administrador

No painel do Supabase, vá em Authentication → Users → Add user. Crie com o seu e-mail e uma senha forte. Esse será o login do painel admin.

  • ✓
    E-mail: o seu e-mail de administradora
  • ✓
    Senha: mínimo 8 caracteres, com letras e números
  • ✓
    Marque "Auto Confirm User" para não precisar confirmar e-mail
Guarde essa senha! Ela é usada para entrar no painel admin em admin/login.html.
6
Hospedar os arquivos (para acesso online)

Para que qualquer pessoa acesse o portal pela internet, você precisa hospedar os arquivos. A opção mais fácil e gratuita é o Netlify:

Acessar netlify.com →
  • ✓
    Crie uma conta gratuita no Netlify
  • ✓
    Clique em Add new site → Deploy manually
  • ✓
    Arraste a pasta inteira do portal para a área indicada
  • ✓
    O Netlify vai gerar um link como pesquisadoras-es.netlify.app
  • ✓
    Você pode depois configurar um domínio próprio (ex: pesquisadorases.com.br)
💡 Enquanto estiver testando, pode abrir os arquivos diretamente no navegador (duplo clique no index.html). As funcionalidades de banco de dados vão funcionar normalmente.
7
Testar tudo
  • ✓
    Abra index.html — o portal público deve carregar (vazio no início)
  • ✓
    Abra admin/login.html — faça login com o e-mail e senha que criou
  • ✓
    No painel, vá em Cadastro com IA e teste cadastrar uma pesquisadora
  • ✓
    Abra inscricao.html e faça uma inscrição de teste
  • ✓
    Volte ao painel, vá em Inscrições e aprove a inscrição de teste
  • ✓
    Atualize o index.html — a pesquisadora aprovada deve aparecer no portal
🎉
